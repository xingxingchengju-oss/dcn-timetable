Assignment 2: Course Timetable Inquiry System 
 
I. Introduction 
Modern educational institutions rely on efficient digital systems to manage and deliver 
course information. A course timetable inquiry system enables students to conveniently 
access up-to-date scheduling information, while administrators can maintain and update 
records dynamically. 
In this assignment, you will design and implement a client-server-based timetable inquiry 
system using Windows Socket Programming in C++. The system must support multiple 
concurrent users and provide reliable access to structured timetable data stored in a 
backend database.

V. Non-Functional Requirements 
Your system should be capable of handling invalid or malformed requests gracefully, 
preventing crashes caused by unexpected input, and ensuring a reasonable response time 
even under concurrent access. In addition, it should maintain a clean and modular code 
structure to support readability, maintainability, and future extensibility. 
 
VI. Bonus Features 
You may implement additional features for extra credit, such as developing a graphical
